# Dragon AI Image - Docker Self-Hosted Deployment

This directory contains everything needed to run Dragon AI Image on your own server using Docker.

## Quick Start

```bash
# 1. Run the setup script
bash scripts/setup.sh

# 2. Edit configuration
nano .env

# 3. Start services
docker-compose up -d

# 4. Check status
docker-compose ps
```

## What's Included

### Docker Files
- **Dockerfile**: Multi-stage build for Node.js application
- **docker-compose.yml**: Complete stack with MySQL, Node.js app, and Nginx

### Scripts
- **scripts/setup.sh**: One-click setup script (recommended)
- **scripts/backup.sh**: Database and application backup
- **scripts/monitor.sh**: Health monitoring and uptime checks

### Configuration
- **nginx/nginx.conf**: Nginx reverse proxy configuration
- **SELF_HOSTED_GUIDE.md**: Comprehensive deployment guide
- **.env.example**: Environment variables template (copy to .env)

### Documentation
- **SELF_HOSTED_GUIDE.md**: Complete installation and configuration guide

## System Requirements

- **Docker**: 20.10+
- **Docker Compose**: 1.29+
- **OS**: Ubuntu 22.04 LTS (recommended) or any Linux with Docker
- **RAM**: 2GB minimum, 4GB+ recommended
- **Storage**: 20GB minimum, 50GB+ recommended

## Configuration

### Environment Variables

Copy `.env.example` to `.env` and update:

```bash
cp .env.example .env
nano .env
```

Key variables:
- `DATABASE_URL`: MySQL connection string
- `JWT_SECRET`: Session signing secret
- `VITE_APP_ID`: Firebase project ID
- `SMTP_*`: Email configuration
- `ALERT_EMAIL`: Alert recipient

### SSL Certificates

Self-signed certificates are generated by setup script. For production:

```bash
# Use Let's Encrypt
sudo certbot certonly --standalone -d your-domain.com
cp /etc/letsencrypt/live/your-domain.com/fullchain.pem ssl/cert.pem
cp /etc/letsencrypt/live/your-domain.com/privkey.pem ssl/key.pem
docker-compose restart nginx
```

## Troubleshooting

### Services won't start
```bash
docker-compose logs
docker-compose build --no-cache
docker-compose up -d
```

### Database connection failed
```bash
docker-compose logs mysql
docker-compose exec mysql mysql -u root -p$MYSQL_ROOT_PASSWORD -e "SELECT 1"
```

### High memory usage
```bash
docker stats
docker-compose restart app
```

### SSL certificate issues
```bash
openssl x509 -in ssl/cert.pem -text -noout
```

## Performance Tuning

### Increase Resources
Edit `docker-compose.yml`:
```yaml
services:
  app:
    deploy:
      resources:
        limits:
          cpus: '2'
          memory: 2G
        reservations:
          cpus: '1'
          memory: 1G
```

### Database Optimization
```bash
docker-compose exec mysql mysql -u root -p$MYSQL_ROOT_PASSWORD
> SET GLOBAL max_connections = 1000;
> SET GLOBAL innodb_buffer_pool_size = 1G;
```

## Backup Strategy

### Daily Backups
Automated at 2 AM (configurable in crontab)

### Manual Backup
```bash
bash scripts/backup.sh
```

### Restore
```bash
docker-compose down
gunzip < backups/mysql_backup_*.sql.gz | \
  docker-compose exec -T mysql mysql -u root -p$MYSQL_ROOT_PASSWORD dragon_ai_image
docker-compose up -d
```
